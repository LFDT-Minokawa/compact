# `tests-e2e-rust` вЂ” cross-language byte-parity tests

This crate validates `compactc --rust` output against the existing
TypeScript backend. For every `.compact` example under
[`../examples/`](../examples/), there's:

- a Rust crate under `contracts/<name>-fixture/` (the codegen output),
- a TS reference capture under `fixtures/<name>-ts-state.json`,
- a `cargo test` under `tests/<name>_fixture.rs` that drives the Rust
  contract and asserts the serialized `ContractState` matches the TS
  capture byte-for-byte.

A new contributor's mental model: **TS is the spec, Rust is the
candidate, the JSON is the contract.** If a Rust test fails, either
the Scheme codegen or `compact-runtime` (or both) produced bytes that
don't match `ContractState.serialize()` from `@midnight-ntwrk/compact-runtime`.

## Adding a new fixture

The pattern is the same every time. The walked steps below produce one
new commit-worth of work.

### 1. Author the Compact source

Drop a minimal example under [`../examples/`](../examples/):

```compact
// examples/my_feature_fixture.compact
import CompactStandardLibrary;

export ledger c: Counter;

constructor() {
  // exercise the new feature here
}

export circuit ping(): [] {
  c.increment(0);
}
```

Keep fixtures tiny вЂ” one ledger field per concept being tested. The
`ping` circuit is a no-op that ensures at least one exported circuit
exists (TS-side `initialState()` registers per-circuit operation map
entries, so an empty exports list breaks `ContractState` equality).

### 2. Generate the Rust crate

```bash
./result/bin/compactc --skip-zk examples/my_feature_fixture.compact \
  /tmp/my-feature-driver/
```

This produces `/tmp/my-feature-driver/contract/{lib.rs,Cargo.toml}`.
Copy them into a new crate directory:

```bash
mkdir -p tests-e2e-rust/contracts/my-feature-fixture/
cp /tmp/my-feature-driver/contract/lib.rs   tests-e2e-rust/contracts/my-feature-fixture/lib.rs
cp /tmp/my-feature-driver/contract/Cargo.toml tests-e2e-rust/contracts/my-feature-fixture/Cargo.toml
```

Then edit `Cargo.toml`'s `package.name` to `compact-contract-my-feature-fixture`
and the workspace `Cargo.toml` to add the new crate as a workspace member.

### 3. Capture the TS reference state

Create `fixtures/capture-my-feature-fixture.mjs`, mirroring an existing
small fixture like `fixtures/capture-list-fixture.mjs`. The script:

```javascript
import { Contract } from '/tmp/my-feature-driver/contract/index.js';
import * as cr from '@midnight-ntwrk/compact-runtime';

const witnesses = {};
const contract = new Contract(witnesses);

const emptyCpk = { bytes: new Uint8Array(32) };
const constructorCtx = {
  initialPrivateState: null,
  initialZswapLocalState: cr.emptyZswapLocalState(emptyCpk),
};

const initResult = contract.initialState(constructorCtx);
const afterInitHex = Buffer.from(initResult.currentContractState.serialize()).toString('hex');

process.stdout.write(JSON.stringify({ afterInit: { stateHex: afterInitHex } }, null, 2) + '\n');
```

Run it (the TS-side `compactc` needs to have produced an importable
JS contract вЂ” re-run the same `compactc` invocation against a TS
output directory):

```bash
compactc --skip-zk examples/my_feature_fixture.compact /tmp/my-feature-driver/
echo '{"type":"module"}' > /tmp/my-feature-driver/contract/package.json
ln -sfn "$PWD/node_modules" /tmp/my-feature-driver/contract/node_modules

node tests-e2e-rust/fixtures/capture-my-feature-fixture.mjs \
  > tests-e2e-rust/fixtures/my-feature-fixture-ts-state.json
```

### 4. Write the Rust byte-parity test

Mirror `tests/list_fixture.rs`. The pattern:

```rust
use compact_contract_my_feature_fixture::Contract;
use compact_runtime::*;
use midnight_serialize::tagged_serialize;
use midnight_storage::storage::HashMap;
use tests_e2e_rust::SmallFixtureTsReference;

fn fixture() -> SmallFixtureTsReference {
    SmallFixtureTsReference::load(concat!(
        env!("CARGO_MANIFEST_DIR"),
        "/fixtures/my-feature-fixture-ts-state.json"
    ))
}

#[test]
fn my_feature_fixture_init_byte_parity() {
    let ts_ref = fixture();
    let contract: Contract<(), NoWitnesses> = Contract::new(NoWitnesses);
    let result = contract.initial_state(ctor_ctx()).expect("initial_state");

    let envelope = make_envelope(result.current_contract_state.clone());
    let mut buf = Vec::new();
    tagged_serialize(&envelope, &mut buf).expect("tagged_serialize");

    assert_eq!(buf, ts_ref.after_init.state_bytes(), "byte-parity");
}
```

(`ctor_ctx`, `make_envelope` are tiny test helpers вЂ” copy from a
neighbouring test file.)

### 5. Add the dev-dep + run

```toml
# tests-e2e-rust/Cargo.toml [dev-dependencies]
compact-contract-my-feature-fixture = { path = "contracts/my-feature-fixture" }
```

```bash
cargo test -p tests-e2e-rust --test my_feature_fixture
```

A passing test means the Rust crate produced the same bytes as the TS
backend. If it fails, run with `--nocapture` to see the byte diff in
the assertion message вЂ” the offset usually points straight at the
encoding mismatch.

### 6. Run the regression guard

```bash
cargo test -p tests-e2e-rust --test codegen_regression
```

This re-runs `compactc --rust` against every example and asserts the
emitted `lib.rs` is byte-identical to the committed one. **Run this
after every Scheme edit** вЂ” it catches the failure mode where the
codegen produces broken Scheme that the build hides because the test
suite drives the *committed* `lib.rs`, not freshly-regenerated output.

## When a test fails

| Symptom | Likely cause | Fix |
|---|---|---|
| `assert_eq!(buf, ts_ref.after_init.state_bytes(), вЂ¦)` fails with a small diff | An alignment / encoding mismatch in `runtime-rs` (e.g. `AlignmentAtom::Bytes { length }` off by one). | Read `runtime-rs/src/std_lib/adts.rs` and the upstream `impl Aligned` for the type. |
| `codegen_regression` fails for fixture X | The emitter (`rust-passes-*.ss`) now produces a different `lib.rs` than what's committed. | Either fix the Scheme to restore prior output, or regenerate and commit the new `lib.rs` if the change is intentional. |
| `cargo build -p compact-contract-X` fails | The hand-edited `lib.rs` references an item not in the runtime's prelude. | Re-export the item from `runtime-rs/src/lib.rs`, or fix the hand-edit. |
| TS capture script hangs / throws | The TS witness was unable to satisfy a circuit's preconditions (e.g. `assert(public_key(sk) == authority)`). | Mirror the assertion's input on the TS side; record `error` in the JSON if irrecoverable, and `#[ignore]` the Rust test. |
