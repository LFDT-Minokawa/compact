# `compact-runtime-macros`

[![docs.rs](https://img.shields.io/docsrs/compact-runtime-macros)](https://docs.rs/compact-runtime-macros)
[![License: Apache-2.0](https://img.shields.io/badge/license-Apache--2.0-blue.svg)](../LICENSE)

Procedural macros for [`compact-runtime`](https://crates.io/crates/compact-runtime).

**You almost certainly don't need to depend on this crate directly.** It's
a transitive dependency of `compact-runtime`, re-exported as
`compact_runtime::witnesses` for use by code generated by `compactc --rust`.

## Why a separate crate?

Rust requires procedural macros to live in a `proc-macro = true` crate
that's compiled as a host-side library, separate from the runtime
library that consumes it. So `compact-runtime-macros` exists purely to
satisfy the language's compilation model — its public surface mirrors
what `compact-runtime` re-exports.

## What it exports

| Macro | Purpose |
|---|---|
| `witnesses!` | Synthesises the `Witnesses<PS>` trait impl boilerplate for a hand-written witness module. Generated `compactc --rust` output references the trait directly; users implementing witnesses can use this macro to skip the per-method boilerplate. |

## Versioning

`compact-runtime-macros`'s version is lock-stepped to `compact-runtime`
via an exact-version pin (`compact-runtime-macros = { version =
"=0.16.100" }` in `compact-runtime`'s manifest). Bump them together; do
not publish one without the other.

See the [crates.io publication
plan](../docs/superpowers/notes/2026-06-02-crates-io-publication-plan.md)
for the release procedure.
